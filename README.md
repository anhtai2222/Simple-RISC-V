# Simple-RISC-V

A minimalist 8‑bit RISC‑like processor design, assembler and simulation environment.

This repository contains a simple RISC‑V inspired CPU implemented in Verilog together with a small command‑line assembler written in C++.  The HDL portion targets Xilinx FPGAs (an Arty‑Z7‑20 design is provided), and the assembler produces a memory image for use in simulation or synthesis.

---

## 🔧 Project Structure

```
Simple-RISC-V/
├── assembler/              # C++ sources for a tiny assembler
│   ├── headers/            # header files (parser, utils)
│   ├── sources/            # implementation (.cpp) files
│   └── Makefile            # build script for the assembler
├── RISC.srcs/              # HDL source files and testbenches
│   ├── sources_1/new/      # Verilog modules (CPU, ALU, registers, etc.)
│   └── sim_1/new/          # Verilog test benches
├── RISC.runs/              # Vivado run directories (implementation/synthesis)
├── RISC.sim/               # simulation directories generated by Vivado
├── xdc_files/              # FPGA constraint file (Arty‑Z7‑20 pinout)
└── README.md               # this document
```

## 🚀 Features

- 8‑bit instruction word with a 3‑bit opcode and 5‑bit immediate/register field
- Supported instructions: `hlt`, `skz`, `add`, `and`, `xor`, `lda`, `sto`, `jmp`
- Simple one‑hot controller, register file and ALU in Verilog
- Basic assembler translates human‑readable mnemonics into machine code
- Testbenches provided for unit‑level simulation of components and full CPU
- Vivado project files for synthesis/implementation on Xilinx FPGAs

## 🛠 Building the Assembler

```bash
cd assembler
make             # compiles sources to build/cpu
```

The `cpu` binary accepts a text file containing assembly instructions and produces
`output.mem`, which is a hex memory image suitable for loading into the CPU's
instruction/data memory in simulation or on real hardware.

### Example

```text
$ cat program.asm
add R1
xor R2
jmp 1F
hlt

$ ./build/cpu program.asm
Done. Output filename: output.mem
```

## 📁 Verilog Design and Simulation

The HDL files live under `RISC.srcs/sources_1/new/`.  Key components include:

- `cpu.v` – top‑level CPU integrating the datapath and controller
- `controller.v` – finite state machine generating control signals
- `alu.v` – arithmetic/logic unit (add, and, xor)
- `registerNbits*` – n‑bit register primitives
- Memory modules and multiplexers for the datapath

Test benches for each module can be found in `RISC.srcs/sim_1/new/`.  To run
behavioral simulation using Vivado:

```bash
# launch Vivado from the project root
vivado -mode batch -source run_simulation.tcl
```

*(Replace `run_simulation.tcl` with the appropriate script generated by Vivado
or use the GUI to elaborate and simulate.)*

## 💡 FPGA Implementation

Constraint files for the Xilinx Arty‑Z7‑20 board are in
`xdc_files/Arty-Z7-20-Master.xdc`.  The `RISC.runs/` directories contain
synthesis and implementation reports generated by Vivado; these are not typically
modified by hand.

To build bitstreams:

1. Open `Simple-RISC-V.xpr` in Vivado (if available) or create a new project
   pointing at `RISC.srcs` and `xdc_files`.
2. Run synthesis and implementation.
3. Generate the bitstream and program your FPGA.
